Free and Open Source PaaS-platform for seamless management of virtual machines, managed Kubernetes, and Databases-as-a-Service<\/p>\n\n\n\n

Platform Goals<\/h3>\n\n\n\n
    \n
  • Provide a simple and user-friendly interface for running:\n
      \n
    • Kubernetes clusters<\/li>\n\n\n\n
    • Databases<\/li>\n\n\n\n
    • Virtual machines<\/li>\n\n\n\n
    • Load balancers<\/li>\n\n\n\n
    • HTTP caching service<\/li>\n<\/ul>\n<\/li>\n\n\n\n
    • Provide a development environment as quickly as possible<\/li>\n\n\n\n
    • Make it easy to install and operate a cloud platform<\/li>\n<\/ul>\n\n\n\n

      In other words, it is an alternative to OpenStack that can be installed with just two commands and has all the necessary features right out of the box!<\/em><\/p>\n\n\n\n

      Target audience<\/h3>\n\n\n\n
        \n
      • As an IaaS platform for hosting companies and businesses offering cloud services<\/li>\n\n\n\n
      • As an internal cloud solution for companies of any size with their own server infrastructure<\/li>\n\n\n\n
      • As a fast and cost-effective tool for spawning development environments<\/li>\n<\/ul>\n\n\n\n

        Standardization and no vendor lock-in<\/h3>\n\n\n\n

        The platform is standartised as possible and reuses widely known technologies and tools.

        During integration, the customer is given detailed training on how each of its components works until there is a 100% understanding of its structure and how to develop and maintain the platform independently.<\/p>\n\n\n\n

        Security<\/h3>\n\n\n\n

        By default, tenants are isolated and can only interact with each other after explicitly enabling network peering between them.<\/p>\n\n\n\n

        Monitoring out of the box<\/h3>\n\n\n\n

        Each instance of each service is accompanied by a set of pre-configured dashboards and alerts.<\/p>\n\n\n\n

        Easy to extend<\/h3>\n\n\n\n

        Each package in the platform consists of a set of YAML files. Therefore, anyone with some familiarity with Kubernetes primitives can modify or expand the platform. Delivery of packages to the system is handled by FluxCD, a well-known and widely used tool in the community.<\/p>\n\n\n\n

        Screenshots<\/h3>\n\n\n\n

        Despite the platform having a graphical user interface, the main goal for it is to facilitate a quick dive in and provide a visual demonstration of the platform’s features.<\/p>\n\n\n\n
